Admin Guide

Configuring SETKI Grid for your organization. Last updated: 2026-07-18

1. Opening settings

The settings panel (⚙) appears when you hover over the grid, but only for users whose Zoho CRM Profile is "Administrator" — this is hardcoded, not configurable. Everyone else never sees the icon. Settings are saved to your Zoho CRM organization and apply to every user, not just you. Changes preview live in the grid as you edit — nothing is applied for other users until you click Save.

2. Columns

Pick which fields of the module appear as columns. Hidden columns are removed entirely — not collapsed, not accessible to users. "Select all" / "clear all" toggles are available.

For each visible column you also set whether it's editable — users can change values in editable columns inline; other columns are view-only for everyone, regardless of their Zoho CRM edit permissions on that field.

3. Pinning columns

Choose columns to pin to the left — they stay visible while users scroll horizontally through the rest. Useful for identifying columns (name, stage) on wide tables.

4. Filters and sorting

  • —Set predefined filters on any column — including columns you've chosen not to display. These apply automatically when the grid loads
  • —You can hide the filter row entirely — predefined filters still apply, users just can't change them
  • —Set a predefined sort order the same way, with the same option to hide sorting controls from users while keeping the order enforced

5. General settings

  • —Default rows per page
  • —Totals row — turn on/off, and per numeric column choose Sum, Average, or none
  • —Table height — fixed pixel height or fill the container
  • —Read-only mode — switches the entire grid to view-only for every non-admin user, regardless of the per-column editable setting

6. Number & percent visualization

Available per numeric column, shown only for columns where the field type applies:

  • —Star rating display for numeric fields
  • —Stock mode — replaces the number with an up/down arrow and color
  • —Color thresholds — set an upper and lower bound with custom colors; cells outside the range are highlighted (available for both number and percent columns)
  • —Decimal places and a column suffix (numbers) — configurable per column
  • —Percent columns additionally render as a progress bar and support their own up/down indicator

7. Dates and text

  • —Deadline highlighting for date columns — on by default. Pick the color for "due today" and "overdue" with a color picker (defaults: amber for today, red for overdue)
  • —Avatars for lookup/owner fields — on by default, shows a colored circle with initials next to the name

8. Presets

A preset is a saved snapshot of the full configuration above. Presets are global — any administrator can save, apply, or delete any preset. Applying a preset copies its settings into the current draft (visible in the live preview); nothing is written to your organization until you save.

Typical use: build separate presets per team or per module — e.g. a warehouse view, a sales pipeline view, a read-only view for external contractors — and switch between them in one click instead of reconfiguring manually.

9. Roles — assigning presets per Zoho Profile

The Roles tab maps each Zoho CRM Profile (Administrator, Standard, or any custom profile in your org) to a preset. When a user opens the grid, SETKI checks their Profile and automatically applies the mapped preset — no per-user setup needed, since Zoho already assigns Profiles to users.

  • —Profiles are pulled automatically from your Zoho CRM organization
  • —A profile left as "— no assignment —" falls back to the base configuration (the settings above, not a preset)
  • —A profile marked with a warning icon existed in a previous mapping but no longer exists in your CRM (renamed or deleted) — remove it with the ✕ next to it

This is access within the grid (which columns, filters, edit rights a role sees) — not licensing. It's independent from Free/Pro seat assignment below.

10. JSON editor

Every setting above is also exposed as a single JSON document, editable directly with search and one-click copy. It's two-way: editing a toggle in the GUI updates the JSON immediately, and editing the JSON updates the GUI. Useful for copying a full configuration between widgets, or for AI-assisted config generation.

11. Multiple tables (Tabs)

If one widget placement should show more than one table — for example the same module with different filters for different teams — administrators can add tabs from the sidebar that appears next to the grid:

  • —Click the + tile at the bottom of the sidebar to add a tab
  • —Hover a tab and use the flyout panel — ✎ to rename, 🗑 to delete (the first/default tab can't be deleted)
  • —Each tab has fully independent settings — module, columns, filters, everything in this guide is configured separately per tab

Everyone can switch between existing tabs; only administrators can add, rename, or delete them.

12. Managing access — Free tier and Pro seats

The License tab shows a different view depending on whether your organization has a Pro subscription:

Free tier (no license key entered)

The first 3 users of your organization by account creation date get full access automatically — no configuration needed. This tab just shows you who those 3 are (read-only) and a counter. To extend access beyond 3 users, purchase a Pro subscription and enter the license key here.

Pro tier (license key entered)

  • —A checklist of every user in your org — check the ones who should have Pro access, up to the number of seats you purchased
  • —Saving the key and saving seat assignments are separate actions (separate buttons)
  • —"Refresh limit" re-checks how many seats you've purchased — use this right after buying additional seats
  • —"Remove key" (with confirmation) reverts the organization to the Free tier rules

Seats are purchased separately at setki.dev — billing is handled by LemonSqueezy, not inside Zoho CRM. See the Terms of Service for details.
